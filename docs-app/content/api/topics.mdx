# Topics

A **topic** is a named append-only log. This page covers its lifecycle: creating and
configuring one (`PUT`), reading its current state (`GET`), listing them (`GET`), and
deleting one (`DELETE`). Writing and reading *records* live on [Writing](/api/writing) and
[Reading](/api/reading); this is the control plane.

Topic names match `^[A-Za-z0-9][A-Za-z0-9._:-]{0,254}$` — 1–255 chars, starting alphanumeric,
allowing `. _ : -`. The `:` enables namespacing like `orders:tenantA`. Names are
case-sensitive and byte-exact. A topic is created lazily on first write, so the `PUT` here is
optional — use it to set config up front.

## PUT /v0/topics/:topic — create or configure a topic

<div className="route"><span className="verb put">PUT</span><span className="path">/v0/topics/:topic</span></div>

Create a topic, or update an existing topic's config. Idempotent and upsertable. Requires the
`admin` scope when auth is enabled.

**Request body** — the [Topic config object](/api/conventions#the-topic-config-object). All
fields are optional; an empty `{}` creates an all-default topic. Set `"type":"queue"` to make
this a queue (and supply the queue-tuning fields here too).

```json
{ "ttl_ms": 60000, "cap_records": 1000000, "discard": "old", "durable": true, "priority": 10 }
```

A durable, unbounded `transcode` queue that fails loudly rather than dropping unconsumed work:

```json
{ "type": "queue", "durable": true, "discard": "reject",
  "lease_ms": 30000, "claim_jitter_ms": 0, "max_deliveries": 5, "dead_letter": "transcode.dlq" }
```

```bash
curl -X PUT $TOPICS/v0/topics/orders \
  -H 'content-type: application/json' \
  -d '{ "durable": true, "cap_records": 1000000, "ttl_ms": 0 }'
```

**Behavior**

- Topic **absent** → created with the given config merged over defaults → `201`.
- Topic **exists, identical config** → no-op → `200`.
- Topic **exists, different config** → updated → `200`. Changes apply going forward; they
  never rewrite stored records, but a tightened `cap`/`ttl` makes existing records eligible
  for lazy eviction/expiry.

<Callout type="warning">
  **`type` is the one immutable field.** A `PUT` that changes `type` on an existing topic
  (`log`→`queue` or `queue`→`log`) returns `409 topic_exists_incompatible`. Every other field
  is mutable in `/v0`, applied going forward.
</Callout>

**Response** (`201` on create, `200` on update/no-op)

```json
{
  "topic": "orders",
  "created": true,
  "config": { "type": "log", "ttl_ms": 0, "cap_records": 1000000, "cap_bytes": 0,
              "discard": "old", "durable": true, "durability": "fsync", "priority": null,
              "auto_priority": true, "auto_create": true,
              "idempotency_window_ms": 120000, "dedupe_node": true,
              "lease_ms": 30000, "claim_jitter_ms": 0, "max_deliveries": 0,
              "dead_letter": null, "leases_durable": false },
  "performance": { "server_total_ms": 0.22 }
}
```

| Field | Type | Meaning |
|---|---|---|
| `topic` | string | The topic name. |
| `created` | bool | `true` only when this call brought the topic into existence. |
| `config` | object | The full resolved config, including the queue-tuning fields (inert on a `"log"` topic) and the resolved `durability` class. |

**Errors**

| `error.code` | When |
|---|---|
| `invalid_request` | Bad name, field, or value (e.g. `dead_letter` equal to this topic). |
| `topic_exists_incompatible` | A `PUT` changed `type`, or made another incompatible change. |
| `forbidden` | The key lacks the `admin` scope or the name is outside its prefix allowlist. |

## GET /v0/topics/:topic — get topic state

<div className="route"><span className="verb get">GET</span><span className="path">/v0/topics/:topic</span></div>

Read current state: head seq, earliest retained seq, count, byte size, and config. The cheap
"where am I / how much lag" call. Requires the `read` scope. By default it bumps the topic's
auto-priority recency clock.

**Query** — `touch` (bool, default `true`): set `false` to **not** update auto-priority
recency (for monitoring scrapes that should not make a topic look "hot").

```bash
curl $TOPICS/v0/topics/orders
```

**Response** (`200`) — a plain log topic

```json
{
  "topic": "orders",
  "type": "log",
  "head_seq": 480231,
  "earliest_seq": 479101,
  "next_seq": 480232,
  "count": 1130,
  "bytes": 2310912,
  "config": { "...": "..." },
  "effective_priority": 500,
  "last_write_ts": 1748450000123,
  "last_read_ts": 1748450009000,
  "performance": { "server_total_ms": 0.05 }
}
```

| Field | Meaning |
|---|---|
| `type` | `"log"` (default, may be omitted) or `"queue"`. |
| `head_seq` | Highest assigned seq (log end). `0` for a fresh empty topic. |
| `earliest_seq` | Seq of the first currently-live record — not evicted, not TTL-expired, **not deleted** (log start). When the topic is empty, `earliest_seq = head_seq + 1`. |
| `next_seq` | Seq the next append will receive (`head_seq + 1`). A handy tail cursor: read from `next_seq - 1` to get only new records. |
| `count` / `bytes` | Currently retained records and payload bytes (approximate under lazy eviction). |
| `effective_priority` | The priority the scheduler is using right now (manual, or auto-derived). |
| `last_write_ts` / `last_read_ts` | Recency clocks in ms; `null` if never. |

<Callout type="important">
  Read `earliest_seq` from the server — never compute the retained floor as `head_seq -
  cap_records`. Eviction is segment-granular and lazy (it may transiently exceed the cap),
  and deletes advance `earliest_seq` independently. A consumer whose `from_seq + 1 <
  earliest_seq` has fallen below the floor; it gets a tombstone only when the loss was
  **involuntary** (cap/TTL), otherwise the cursor advances silently past deleted seqs. See
  [Tombstones](/guarantees/tombstones).
</Callout>

### The `queue` sub-object

A **queue** topic (`type:"queue"`) additionally returns a `queue` sub-object with live
counters:

```json
{
  "topic": "transcode",
  "type": "queue",
  "head_seq": 480231, "earliest_seq": 479101, "next_seq": 480232,
  "count": 1130, "bytes": 2310912,
  "config": { "type": "queue", "lease_ms": 30000, "...": "..." },
  "queue": { "ready": 842, "in_flight": 288, "dead_lettered": 4 },
  "effective_priority": 500,
  "last_write_ts": 1748450000123, "last_read_ts": 1748450009000,
  "performance": { "server_total_ms": 0.06 }
}
```

| `queue` field | Meaning |
|---|---|
| `ready` | Jobs claimable right now. |
| `in_flight` | Jobs currently leased to a claimer (awaiting ack / reclaim). |
| `dead_lettered` | Jobs moved to the dead-letter topic. |

The `queue` sub-object is **absent** on a `"log"` topic. See [Queues](/api/queues) for the
claim / ack / nack / extend lifecycle behind these counters.

**Errors**

| `error.code` | When |
|---|---|
| `topic_not_found` | The topic does not exist (a state read **never** auto-creates). |
| `forbidden` | The key lacks the `read` scope or the name is outside its prefix allowlist. |

## GET /v0/topics — list topics

<div className="route"><span className="verb get">GET</span><span className="path">/v0/topics</span></div>

Enumerate topics with summary state. Opaque-cursor paginated. Requires the `read` scope; a
prefix-restricted key sees only the names inside its allowlist.

**Query**

| Param | Type | Default | Meaning |
|---|---|---|---|
| `prefix` | string | none | Only topics whose name starts with this prefix. |
| `page_size` | int | `100` | Page size, max `1000`. |
| `cursor` | string | none | Opaque continuation cursor from a prior page's `next_cursor`. |
| `touch` | bool | `false` | Listing does not bump auto-priority unless set `true`. |

```bash
curl "$TOPICS/v0/topics?prefix=orders&page_size=100"
```

**Response** (`200`)

```json
{
  "topics": [
    { "topic": "orders", "head_seq": 480231, "earliest_seq": 479101, "count": 1130, "bytes": 2310912, "durable": true,  "effective_priority": 500 },
    { "topic": "events", "head_seq": 99,     "earliest_seq": 1,      "count": 99,   "bytes": 8112,    "durable": false, "effective_priority": 3 }
  ],
  "next_cursor": "eyJhZnRlciI6Im9yZGVycyJ9",
  "performance": { "server_total_ms": 0.18 }
}
```

| Field | Meaning |
|---|---|
| `topics[]` | Summary state per topic (a subset of the full topic-state object). |
| `next_cursor` | Opaque continuation cursor. **Omitted on the final page** — its absence means done. Pass it back as `?cursor=`. |

**Errors**

| `error.code` | When |
|---|---|
| `invalid_request` | Malformed or corrupt `cursor`. |
| `forbidden` | The key lacks the `read` scope. |

## DELETE /v0/topics/:topic — delete a topic

<div className="route"><span className="verb delete">DELETE</span><span className="path">/v0/topics/:topic</span></div>

Permanently remove a topic and **all** its records, dedupe state, the per-topic tag index, and
every router referencing it as `source` or `dest`. Idempotent and irreversible. Requires the
`delete` scope.

<Callout type="warning">
  This deletes the whole topic, not records within it. To remove records by seq range or tag
  while keeping the topic, use [`POST /v0/topics/:topic/delete`](/api/deletion) instead.
</Callout>

**Query** — `if_empty` (bool, default `false`): when `true`, delete only if `count == 0`,
else `409 topic_not_empty`.

```bash
curl -X DELETE $TOPICS/v0/topics/orders
```

**Response** (`200`)

```json
{ "topic": "orders", "deleted": true,
  "routers_removed": ["orders->audit", "orders->analytics"],
  "performance": { "server_total_ms": 0.31 } }
```

| Field | Meaning |
|---|---|
| `deleted` | `true` if the topic existed and was removed; `false` if it was already absent. |
| `routers_removed` | Names of routers cascaded away by this delete (those referencing the topic). Empty when none. |

Deleting an absent topic returns `200` with `"deleted": false` and an empty `routers_removed` —
the idempotent path. A later lazy-create makes a **new, empty** topic (seq restarts at the
beginning); stale consumers detect this via the `recreated` tombstone on their next read.

**Errors**

| `error.code` | When |
|---|---|
| `topic_not_empty` | `if_empty=true` and the topic still holds records. |
| `forbidden` | The key lacks the `delete` scope or the name is outside its prefix allowlist. |
