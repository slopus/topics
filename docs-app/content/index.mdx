# A persistent event engine in a <span className="grad">single binary</span>

<p className="hero-sub">
**streams** is an append-only log service over a clean, JSON-first HTTP API. Write
events to named **boxes**, read them back by **sequence**, fan them out with

static binary on one machine, backed by a write-ahead log on local NVMe. Data loss is
always **explicit, never silent**.

```bash
# Write a record (the server assigns its $seq). The box is created on first write.
curl -X POST localhost:4000/v0/boxes/jobs \
  -H 'content-type: application/json' \
  -d '{ "records": [ { "data": { "url": "s3://b/a.png" }, "tag": "job-9001" } ] }'
# → { "box": "jobs", "first_seq": 1, "last_seq": 1, "seqs": [1], "head_seq": 1, ... }

# Read everything after a cursor. The monotonic seq IS the cursor.
curl -X POST localhost:4000/v0/boxes/jobs/diff -d '{ "from_seq": 0 }'
# → { "records": [ { "$seq": 1, "$ts": 1748470000123, "data": { "url": "s3://b/a.png" } } ],
#     "next_from_seq": 1, "head_seq": 1, "caught_up": true, "tombstone": null }
```

<span className="section-eyebrow">Mental model</span>
<h2 className="section-title">Five concepts. That's the whole product.</h2>
<p className="section-lede">
streams is deliberately small. Learn five nouns and you know the system — every
endpoint is one of these doing its job.
</p>

<div className="concept-grid">
  <div className="concept"><div className="c-name">Box</div><p className="c-desc">A named, append-only log of records ordered by a monotonic <code>seq</code>. Created lazily on first write.</p></div>
  <div className="concept"><div className="c-name">Record</div><p className="c-desc">One immutable event. The server assigns <code>$seq</code> and <code>$ts</code>; you supply opaque <code>data</code>, an optional <code>tag</code> and <code>node</code>.</p></div>
  <div className="concept"><div className="c-name">seq</div><p className="c-desc">The cursor. Reads are “give me everything after <code>from_seq</code>.” The client owns its position; advancing it is the ack.</p></div>
  <div className="concept"><div className="c-name">Router</div><p className="c-desc">A server-side <code>source → dest</code> forwarding rule. Fan-out that preserves origin node, so N nodes mirror without echo.</p></div>
  <div className="concept"><div className="c-name">Tombstone</div><p className="c-desc">The explicit “you missed data” signal. Cap/TTL loss returns an in-band gap range — at HTTP 200, never silently.</p></div>
</div>

<section className="section">

<span className="section-eyebrow">The central tradeoff</span>
<h2 className="section-title">Durability is per box — each box buys exactly the guarantee it needs</h2>
<p className="section-lede">
A best-effort cache, a group-committed pub/sub feed, and an fsync-gated job queue
coexist in one process. Routers bridge them, honoring the destination box's class.
</p>

<div className="dura-grid">
  <div className="dura">
    <span className="d-tag">memory</span>
    <h4>Disk-like, best-effort, fastest</h4>
    <p>Same group-committed WAL path as <code>disk</code> and fully queryable, but with no durability guarantee: after a restart its records may survive or be lost (config always survives). Never fsync-gated. For caches, scratch, ephemeral fan-out.</p>
  </div>
  <div className="dura">
    <span className="d-tag">disk</span>
    <h4>Group-committed WAL</h4>
    <p>One fsync amortized across a batch of concurrent writers. Survives a crash minus the un-fsynced tail. The pub/sub default.</p>
  </div>
  <div className="dura">
    <span className="d-tag">fsync</span>
    <h4>fsync-gated ack</h4>
    <p>The ack waits for the group fsync. An acknowledged write survives any crash and is recovered by WAL replay. For queues and ledgers.</p>
  </div>
</div>
